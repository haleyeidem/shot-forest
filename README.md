# Shot Forest

An interactive shot chart for the 2025–26 Minnesota Timberwolves — planted as a forest.

Each tree grows where a player takes a shot. The bigger the tree, the more frequent the shot. The brighter the tree, the better that shot goes down compared to the rest of the league.

🌲 **[See it live →](https://haleyeidem.github.io/shot-forest/)**

---

## How to read it

- **Tree size** = shot volume from that hex (power scale so a few big trees don't drown out the small ones, but the busiest spots clearly tower).
- **Tree color** = the player's FG% from that hex vs. the league's FG% from the *same hex* — a spatial comparison, not a zone-level one. Five-step green ramp from cold to elite.
- **Hex grid** = the same bins as a Goldsberry-style hex chart, rendered as the iconic Timberwolves pine.

The toggle next to the player switcher flips between **Regular Season**, **Playoffs**, and **Full Season** (whichever the latest data pull contains).

---

## Methodology

- **Source.** NBA Stats `ShotChartDetail` via `nba_api`, season `2025-26`, team `Minnesota Timberwolves`.
- **Bins.** Hexagonal grid, radius ≈ 2.5 ft (25.5 court units), full half-court.
- **League baseline.** For each hex, the league's FG% is computed across all NBA players' shots in that same hex this season. The "vs league" delta is a spatial comparison — not "all corner threes," not "all paint shots," but *that exact spot*.
- **Low volume.** Hexes with fewer than 2 attempts are dropped to keep the chart honest.
- **Stylization.** The chart raises the visual baseline to sit just behind the rim so trees plant where the eye expects them; all other court proportions remain faithful to NBA dimensions.

---

## Stack

Single static `index.html` — markup, styles, and the React component live side-by-side so the chart, copy, and data wiring stay in one place. The shot chart is hand-rolled SVG (no D3). One static JSON file (`data/shot-forest.json`, generated by `tools/build_data.py`) is fetched on page load. No backend, no server, no API. Hosted on GitHub Pages.

`support.js` is a third-party runtime that parses the `<x-dc>` block — vendor file, not authored work.
